1. 项目概览

一句话定位

全栈项目，负责小程序前端架构、模型集成、三级定价后端逻辑，优化分包加载性能

时间周期：2024.05 - 2024.07

我的角色：第一开发人

2. 技术架构

系统架构

微信小程序 + Java后端 + MySQL数据库 + BiLG-BLS预测模型，采用分包加载架构优化性能

技术栈

前端：微信原生小程序 后端：Java 数据库：MySQL 图表：ECharts AI模型：BiLG-BLS

3. 前端实现

页面结构

采用Tab架构：首页/商务/购物车/个人中心 + 分包加载数字化服务模块

组件设计

封装通用的商品卡片、图表组件、设备管理组件，提高代码复用率

⚡ 性能优化：分包加载策略，主包只包含首屏必需，将数字化服务等低频模块分离到分包

4. 后端实现

数据库设计

核心表：用户表、设备表、订单表、能耗数据表，设计索引优化查询性能

API 设计

RESTful接口规范，核心接口：设备数据上传、能耗统计、预测结果获取、订单管理

认证权限

基于微信登录的会话管理，用户-企业-设备三级权限体系

BiLG-BLS模型：双向长短期记忆网络 + 宽度学习系统，用于能耗预测和异常检测

关键代码思路

模型实时预测，通过Websocket推送数据到前端，实现仪表盘实时更新

📈 优化点：模型轻量化部署，通过模型剪枝和量化，提升预测速度30%
