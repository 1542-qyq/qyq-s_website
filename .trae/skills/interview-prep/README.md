# interview-prep

一个面试准备 Skill，帮你快速生成一份**针对具体岗位 + 具体公司**的面试准备文档——含公司调研、JD 解读、定制自我介绍、STAR 项目预案、高频问题、反问清单、Gap 清单等 8 个章节。

> 兼容支持 Claude Skill 格式的 agent 工具，包括 Claude Code、Claude.ai、Cowork 等。

> 不是通用模板。每一次都会基于你的简历和目标岗位 JD 重新生成，逐条做证据驱动的匹配分析。

---

## 适用人群

**岗位**：产品 / 商业类岗位

| 方向 | 专项题侧重 |
|------|----------|
| 产品经理（PM） | 产品设计、需求优先级、指标体系、竞品分析 |
| 增长 / 数据产品 | AARRR 漏斗、A/B 实验、数据拆解、渠道质量 |
| 产品运营 | 用户分层、活动设计、内容/社区运营、商业化 |
| 商业策略 / BD | 战略分析、合作谈判、商业模式设计 |

**地区**：当前版本针对**中国大陆求职市场**优化（面经来源：小红书、牛客网、脉脉；公司调研：36氪、虎嗅、App Store 中区；文档默认中文）。海外岗位也能用，但调研深度会受限。

## 安装

### Claude Code

把仓库 clone 到 skills 目录：

```bash
cd ~/.claude/skills
git clone https://github.com/lan1177/interview-prep.git
```

下次启动 Claude Code 时会自动识别。

**更新到最新版**：

```bash
cd ~/.claude/skills/interview-prep
git pull
```

### Claude.ai / 其他支持 Skill 的工具

下载本仓库的 zip 包（或 `git clone`），按你的工具说明导入：

- **Claude.ai**：在 Skills 管理界面上传 `SKILL.md`（含 `references/` 目录一起打包上传）
- **Cowork**：参考 Cowork 的 skill 安装指引，把整个文件夹放到对应位置
- **其他工具**：参考该工具的 skill 安装文档，本仓库结构遵循标准 Claude Skill 格式

> 不论用哪个工具，只要它支持 Claude Skill 格式（含 `SKILL.md` + YAML frontmatter + `references/` 子目录的约定），都能用这个 skill。

## 使用方式

直接告诉 Claude 你要面试，skill 会自动触发：

```
帮我做面试准备，我明天要面字节跳动的抖音产品经理
```

```
我要去面试了，岗位是增长 PM，公司美团，JD 我粘贴给你 [粘贴 JD]
```

skill 会一次性问完简历、JD、岗位方向、面试时间等信息，然后自动调研并生成文档。

## 核心特性

### 🚀 三种速度模式，按面试紧迫度自动切换

| 模式 | 触发条件 | 内容 | 大概耗时 |
|------|---------|------|---------|
| **速准版** | 今天 / 明天就面 | 自我介绍 + STAR + 高频题 + Gap，跳过深度调研 | ~3 分钟 |
| **标准版** | 3-7 天后 | 完整 8 章文档 + 公司/竞品/面经调研 | ~8 分钟 |
| **深研版** | 1 周以上 | 标准版 + 详细竞品对比 + 10 条反问 + 延伸阅读 | ~12 分钟 |

### 🎯 模块模式：只要某一节也行

不想跑完整流程？直接说：

- "帮我写个自我介绍" → 只生成自我介绍
- "可能会问什么问题" → 只生成高频问题
- "我的简历和 JD 差距在哪" → 只生成 Gap 分析

### 📂 灵活输出

- **默认**：直接打印在对话里（所有平台通用，包括 Claude.ai）
- **可选**：保存为本地 Markdown 文件
- **可选**：两者都要

### 🔍 证据驱动的匹配分析

JD 逐条匹配分析必须**先引用简历证据再下判断**，没有证据的条目标 `—` 而非强行打分。避免"我这里都挺匹配的"的假象。

### 📌 真实链接，可追溯

所有来自网络调研的信息都附原始 URL，可点击核实。不编造来源。

## 输出文档结构

完整文档（标准版）包含 8 个部分：

```
零、JD 原文（完整保留）
一、公司与产品调研（公司背景、产品特性、竞品格局、团队文化、小红书热帖、面经总结）
二、JD 深度解读（核心意图、职责白话解读、逐条匹配分析）
三、定制版自我介绍（≤ 250 字，针对该岗位编排）
四、项目深挖问答（3 个 STAR 项目 + 追问预案）
五、高频面试问题与参考回答（含岗位专项题）
六、针对该产品的设计思考
七、反问清单（5-8 条）
附、Gap 清单（按优先级 1/2/3 分组，含 checkbox 和补课建议）
```

示例片段（来自一次真实运行，节选）：

> | JD 要求 | 简历依据 | 匹配度 | gap / 风险 | 面试具体策略 |
> |--------|---------|--------|-----------|------------|
> | 有社区/UGC产品经验者优先 | 「异步捉迷藏UGC系统：日参与度8万，同类玩法4倍」 | ✅强 | — | 用捉迷藏案例详细讲冷启动方案和分发算法设计 |
> | 深度研究用户需求 | 简历未见直接证据 | — | 被追问"你怎么做用户研究"时可能答不具体 | 面试时主动构建叙事：「小红书账号运营是天然的用户研究」 |

## 依赖工具

### 必须（主流 Claude agent 工具均内置）
- `WebSearch` / `WebFetch`：网络调研
- `Read` / `Write`：读简历/JD，保存文档

### 可选（装了的话调研质量会更高）
- [`web-access`](https://github.com/anthropics/claude-code-skills)：访问需登录页面（小红书、脉脉）
- `mcp__xiaohongshu-mcp`：小红书专项 MCP
- [`obsidian-cli`](https://github.com/anthropics/claude-code-skills)：从 Obsidian 知识库读简历/JD

**完全不装可选工具也能完整跑完，只是小红书内容可能拿不到完整页面。**
